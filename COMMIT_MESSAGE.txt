## Architecture Decisions

### 1. fsnotify for FIM
Rationale: Cross-platform (Linux inotify, Windows FSEvents),
battle-tested (used by Docker, Kubernetes), low overhead vs polling.

### 2. BFS for Attack Path Computation
Rationale: Finds shortest paths first (most likely attack vectors),
prevents infinite loops (max depth 5), efficient for < 1000 node graphs.

### 3. External Scanners for SBOM
Rationale: Leverage Anchore/Aqua expertise (Syft/Trivy/Grype),
broad ecosystem support (OCI, SPDX, CycloneDX), active CVE updates.
