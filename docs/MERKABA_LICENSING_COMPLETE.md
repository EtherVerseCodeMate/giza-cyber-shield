## 3. Merkaba Tri-Polar Billing Model

The Merkaba's dual-tetrahedron structure maps to three revenue dimensions, each with its own pricing formula.

### Sun Polarity ☀️ (Active Threats / Consumption-Based)

**The Spinning Sun**: Forward-spinning tetrahedron represents active, dynamic threats being detected.

```
SUN REVENUE = (Scans × $0.10) + (Critical × $1.00) + (High × $0.25) + (Medium × $0.05)

Example:
├─ 200 scans × $0.10      = $20.00
├─ 5 critical × $1.00     = $5.00
├─ 15 high × $0.25        = $3.75
└─ 40 medium × $0.05      = $2.00
                          ─────────
                    SUM:  $30.75
```

**Deity**: Ra (Sun god) — Pay for what harms you  
**Meaning**: Active threats detected during the month

### Earth Polarity 🌍 (Assets Protected / Seat-Based)

**The Grounded Earth**: Reverse-spinning tetrahedron represents stable, protective assets and controls.

```
EARTH REVENUE = (Nodes × $50) + (SANs × $5) + (Instances × $25) + (Services × $10)

Example:
├─ 3 active nodes × $50          = $150.00
├─ 2 Subject Alt Names × $5      = $10.00
├─ 5 VM instances × $25          = $125.00
└─ 10 microservices × $10        = $100.00
                                 ─────────
                           SUM:  $385.00
```

**Deity**: Geb (Earth god) / Asaase (West African) — Protection you maintain  
**Meaning**: Number of assets/infrastructure under protection

### Seed Polarity ⚪ (Compliance Proof / Storage-Based)

**The Still Seed**: Stillness between spins represents archival, evidence, and eternal records.

```
SEED REVENUE = (GB-Days × $0.01) + (Reports × $10) + (Archive GB × $0.005)

Example:
├─ 500 GB-days × $0.01     = $5.00
├─ 10 compliance reports × $10  = $100.00
└─ 1000 archive GB × $0.005    = $5.00
                           ────────
                     SUM: $110.00
```

**Deity**: Thoth (Record keeper) / Shu (Evidence holder) — Proof of compliance  
**Meaning**: Historical data retained for audit trails

### Unified Billing Formula

```
MONTHLY BILL = BASE TIER + SUN + EARTH + SEED

Where:
  BASE TIER = Tier cost ($50-$2000 or custom)
  SUN       = Threat-based consumption
  EARTH     = Asset-based seats
  SEED      = Storage-based retention

Example (Hunter Tier):
├─ Base tier:      $500.00
```

**The Ma'at Balance**: Like the scales of Ma'at in the Egyptian weighing of the heart, customers pay proportionally for:
- **What harms them** (Sun = threats)
- **What shields them** (Earth = assets)

## 5. Air-Gapped Licensing (Shu Breath)

**Pharaoh tier only**: Organizations can operate offline with renewed licensing signatures.

### The Shu Mechanism

In Egyptian mythology, **Shu** is the god of wind/air who lifted the sky from the earth. In Khepra:
- **Shu's Breath** = Licensing signature valid for 365 days (1 solar cycle)
- Allows Pharaoh customers to operate completely air-gapped
- No internet connectivity required (except annual renewal)

### Generation Process

```bash
# Step 1: Online enrollment
adinkhepra enroll \
  --tier pharaoh \
  --license-id lic_pharaoh_dod_001 \
  --organization "DoD Unit X" \
  --generate-offline-key

Outputs:
├─ khepra.pharaoh.key  (encrypted, stored securely)
└─ shu_breath.sig      (offline license signature, 365 days valid)
```

### Transfer to Air-Gapped System

```bash
# Step 2: Secure transfer (not over network)
scp shu_breath.sig user@airgapped:/opt/khepra/licenses/

# Step 3: Offline validation
adinkhepra validate-offline-license \
  --sig-file /opt/khepra/licenses/shu_breath.sig

Output:
  ✅ License valid until: 2027-01-19
  ✅ All Sephirot levels accessible
  ✅ Unlimited node quota
  ✅ System is properly sealed for air-gap
```

### Annual Renewal

```bash
# Once per year: briefly connect to internet
```

```go
// In pkg/license/egyptian_tiers.go

func GenerateOfflineLicense(lic *License) (*OfflineLicense, error) {
    if lic.Tier != TierOsiris {
        return nil, errors.New("offline licensing only for Pharaoh tier")
    }
    
    // Create 365-day valid signature using Dilithium
    now := time.Now()
    expiresAt := now.AddDate(1, 0, 0) // 1 year
    
    payload := fmt.Sprintf("%s|%s|%d|%s", 
        lic.ID, TierOsiris, -1, expiresAt.Format(time.RFC3339))
    
    sig, err := adinkra.Sign(privateKey, []byte(payload))
    if err != nil {
        return nil, err
    }
    
    return &OfflineLicense{
        LicenseID:  lic.ID,
        Signature:  hex.EncodeToString(sig),
        ExpiresAt:  expiresAt,
        ValidDays:  365,
        ShuBreath:  true,  // Mark as Shu Breath
        IssuedAt:   now,
    }, nil
}

func ValidateOfflineLicense(sigFile string) (bool, error) {
    // Read signature from file
    sig, _ := ioutil.ReadFile(sigFile)
    
    // Verify against embedded root CA (no network needed)
    return adinkra.Verify(rootPublicKey, sig), nil
}
```

## 6. Integration with DAG System

### Node Creation with License Enforcement

```go
// In pkg/dag/dag.go

func (m *Memory) Add(n *Node, parents []string, licenseID string) error {
    m.mu.Lock()
    defer m.mu.Unlock()
    
    // ENFORCEMENT STEP 1: Check license quota
    lic, err := licenseManager.GetLicense(licenseID)
    if err != nil {
        return fmt.Errorf("invalid license: %w", err)
    }
    
    // ENFORCEMENT STEP 2: Check node quota
    if lic.NodeCount >= lic.NodeQuota && lic.Tier != TierOsiris {
        return fmt.Errorf("license %s has reached node quota (%d/%d)",
            licenseID, lic.NodeCount, lic.NodeQuota)
    }
    
    // ENFORCEMENT STEP 3: Check Sephirot access for this node type
    sephirotLevel := GetSephirotLevel(n.Type)
    canAccess := false
    for _, level := range lic.SephirotAccess {
        if level >= sephirotLevel {
            canAccess = true
            break
        }
    }
    
    if !canAccess {
        deity := GetRequiredDeity(n.Type)
        return fmt.Errorf("node type %s requires %s tier (%s authority)",
            n.Type, GetMinimumTier(n.Type), deity)
    }
    
    // ENFORCEMENT STEP 4: Add node
    n.ID = n.ComputeHash()
    m.nodes[n.ID] = n
    
    // ENFORCEMENT STEP 5: Register with license
    licenseManager.RegisterNodeCreation(licenseID, n.ID, sephirotLevel)
    
    return nil
}
```

## 8. FAQ

**Q: What happens if I hit my node quota?**  
A: Node creation fails with "quota reached" error. Upgrade your license or remediate/archive existing nodes.

**Q: Can I go air-gapped with Hunter tier?**  
A: No, air-gap licensing (Shu Breath) is Pharaoh-only. Hunter/Hive require internet connectivity.

**Q: What's the difference between Atum and Osiris?**  
A: Atum = $2k/mo fixed, 10 nodes, 30-day retention. Osiris = custom pricing, unlimited nodes, 365-day retention, DoD features.

**Q: How do I know if my node is in danger (state 15)?**  
A: Monitor the Egyptian Fate in your dashboard. State 15 triggers "Ammit the Devourer" critical alert.

**Q: Can I downgrade my license?**  
A: No, tiers only upgrade: Khepri → Ra → Atum → Osiris. Downgrades would risk losing access to nodes.

**Q: How is my bill calculated exactly?**  
A: BASE_TIER + (Scans × $0.10) + (Criticals × $1.00) + (Nodes × $50) + (GB-Days × $0.01)
