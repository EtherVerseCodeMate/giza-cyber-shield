# Legal / positioning note — evidence and certification claims

**Not legal advice.** Use with counsel for customer-facing contracts, RFPs, and regulated markets.

## Principles

1. **Avoid absolute guarantees** (“passes CMMC,” “certified compliant”) unless tied to a defined scope, assessor, and evidence artifact.
2. **Describe outputs accurately:** scans surface signals and configuration findings; **ADINKHEPRA** attestation binds stated posture to cryptographic proof **when certification is purchased and scope is met**.
3. **Third-party programs (CMMC, IronBank, etc.):** Position ASAF as **readiness and evidence tooling**; final authorization remains with the customer’s AO / C3PAO / program office.
4. **IronBank / container hardening:** If you claim “IronBank alignment,” tie claims to **published criteria**, **version**, and **artifact** (e.g. submission checklist, scan report), not generic “approved” language unless you have explicit program sign-off.

## Suggested safe phrasing

- “Structured for **C3PAO / ISSM intake**” (not “guaranteed C3PAO acceptance”).
- “**Evidence-oriented** outputs” / “**traceable** findings” (not “automatic compliance”).
- “**Optional** NemoClaw / agent profile” when scope includes those surfaces.
